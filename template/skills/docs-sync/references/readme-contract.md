# README 同步契约

README 面向首次接触项目、正在判断是否采用以及准备立即试用的读者。保留项目既有语言和风格，不强塞不适用的标准章节。

## 事实顺序

按认知漏斗审计现有结构：

1. 名称与一句具体定位：说明输入、输出或解决的问题，删除空泛营销语。
2. 必要的截图、演示和功能性徽章：链接失效或状态无价值时修复或删除。
3. 可复制的最小 Quick Start：尽量前置，并给出判断成功所需的预期结果。
4. 安装与前置条件：版本、平台、包名和验证命令必须来自当前元数据。
5. 核心用法、配置和限制：从最常用场景开始，诚实说明未支持边界。
6. 详细文档、贡献与许可证：已有独立文档时只保留导航。

项目简单时合并相邻章节；README 很长时把 API、架构解释和贡献流程移到对应文档，保留读者完成首次采用所需内容。

## 同步触发

| 事实变化 | 必查内容 |
|---|---|
| package 描述、定位或对外能力 | 开头描述、特性、限制 |
| CLI/API/配置入口 | Quick Start、使用、参考表、示例 |
| runtime、依赖或发布渠道 | 安装、前置条件、徽章 |
| 顶层结构或架构边界 | 必要的结构摘要与文档链接 |
| CI/release/license | 状态徽章、发布说明、License |

从源码、manifest、CLI `--help`、schema、测试和 CI 枚举真实入口。删除被移除的命令、路径、参数、徽章、链接和旧架构，不保留“未来能力”填补空白。

## 多语言

- 只有项目实际维护镜像时才要求同步。
- 标题顺序、命令、代码块、URL、表格字段和关键约束保持对等；代码实体不翻译。
- 某语言版本无法同步时阻塞该组修改，不能让镜像继续漂移。

## 验证

- 实际运行或静态验证 Quick Start、安装和核心示例。
- 对照代码枚举 CLI/API/配置条目，确认没有漏项或幽灵入口。
- 检查站内链接、图片、徽章和语言切换。
- 逐段删除测试：删掉后不影响采用决策或正确使用的内容应压缩、下沉或删除。

完成标准：新读者可以从 README 判断项目价值、完成最小安装并验证结果；全部声明均有当前事实来源。
