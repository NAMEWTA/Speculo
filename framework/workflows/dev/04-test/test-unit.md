# Test Unit — 单元测试规范

本阶段为本 change 建立**与 PRD AC 一一对照、覆盖边界与异常路径**的单元测试集。AI 在执行 `phase=unit` 时读本文件作为填写指引。

## 一、上下文与产物

- **输入**：同 change 下的 `prd.md`（AC 派生源）、`implement.md`、`design-api.md`
- **产物**：`test-unit.md`（基于 `../_templates/test-unit-template.md`）
- **铁律**：**测试用例必须从 PRD#验收标准 派生**，禁止从实现派生（否则只是"实现已通过"的同义反复）

## 二、章节填写引导

### 测试范围
- 列本 change 涉及的全部**生产代码模块/函数清单**
- 与 `implement.md` 的"实现概要"对齐（无清单 → 测试漏洞）
- 标注哪些模块"已有测试覆盖"（复用既有用例）、哪些"本次新增"

### 用例设计
- 用表格列**关键边界用例 + 异常路径用例**
- 每条用例标注：用例名 / 输入 / 期望输出 / 关联 AC
- 至少覆盖以下 4 类边界：
  - 空 / null / undefined
  - 极大 / 极小 / 边界值
  - Unicode / 特殊字符 / 长字符串
  - 错误路径（异常、失败、超时、并发冲突）

### AC ↔ 测试映射（强制段）

| AC 编号 | 测试类型 | 测试文件:测试名 | 状态 |
|---------|---------|----------------|------|
| AC-1 | unit | src/theme/__tests__/context.test.ts::switches default | ✅ |
| AC-2 | integration | src/theme/__tests__/persist.test.ts::roundtrip | ✅ |
| AC-3 | e2e | （由 test-e2e.md 覆盖） | → e2e |

- 每条 PRD AC 必须在此表出现一次
- 没有任何单测/集成测可覆盖的 AC，标 `→ e2e` 并在 `test-e2e.md` 接续

### 覆盖率
- 跑实际命令并贴出输出（不允许声称"约 80%"）
- 与项目门槛对比（默认 80%；core 模块建议 ≥90%）
- 未达门槛：写"未覆盖路径清单 + 不覆盖理由"（不允许沉默）

## 三、反作弊红线

- 禁止用 mock 屏蔽真实失败：mock 仅用于隔离外部依赖（DB、网络、时间），不用于"让测试通过"
- 禁止删除 / 弱化既有测试来"修复"失败；如确需修改，必须在 PR/commit 说明理由
- bug 修复必须伴随回归测试用例（在用例设计段标 `regression: <bug-id>`）

## 四、推荐 TDD 顺序（非强制）

1. 把 AC 翻译成"失败的测试用例"
2. 跑测试 → 确认失败（fail for the right reason）
3. 写最小实现使其通过
4. 重构（保持绿）

`.speculo/.config/RULES.md` 可启用为强制；本规范默认推荐。

## 五、完成准则（机器可验证）

- `grep -c '\[TODO:' test-unit.md` = 0
- 文件含 `## 测试范围` / `## 用例设计` / `## 覆盖率` 三个标题
- 文件含 "AC" 与 "测试" 至少同时出现的映射表（≥1 行表格）
- 每条 PRD AC 在映射表中出现至少 1 次
- "用例设计" 段含至少 4 类边界关键词（空 / null / 极 / Unicode / 错误 / 异常 / 超时 / 并发 任选 4）
- 覆盖率段含 ` ``` ` 代码块（贴的命令输出）
- `test_coverage` 与 `ac_coverage` 写入 `.status.json`
