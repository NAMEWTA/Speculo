# PRD

## 项目背景

当前用户登录依赖会话 cookie，跨设备登录状态无法同步、登出延迟最高 5 分钟，且无法支持移动 App 与 Web 双端共享登录态。客服反馈每周收到约 30 起"已登出但还能下单"的混乱投诉。本需求把登录态切换为 JWT + refresh token 双 token 方案，统一桌面与移动 App，目标把上述投诉降为 0、并把异地登出延迟降到 ≤30 秒。

## 核心需求

- **US-1（P1）**：作为 Web 用户，我希望关闭浏览器后再打开仍保持登录态，以便不必频繁输密码。
- **US-2（P1）**：作为 App 用户，我希望在某台设备登出后，其他设备 ≤30 秒内同步登出，以便保护账号安全。
- **US-3（P2）**：作为运营管理员，我希望强制下线某个用户的所有 session，以便处理风险账号。
- **US-4（P2）**：作为安全审计员，我希望每次 token 签发与失效都留痕，以便事后追溯。

## 范围与非范围

本次包含：
- JWT 签发与校验流程
- Refresh token 旋转策略
- 跨设备登出广播机制

本次不做：
- SSO（与第三方 IdP 集成），因为需求未覆盖且影响面太大
- 生物识别 / 二次验证，因为本期目标聚焦"状态同步"而非"鉴权强度"

## 验收标准

- **AC-1**（关联 US-1）：**Given** 用户已登录 Web 且关闭浏览器，**When** 1 小时内再打开，**Then** 自动恢复登录态且无需重新输密码。
- **AC-2**（关联 US-2）：**Given** 用户在设备 A 已登录，设备 B 也已登录，**When** 设备 A 触发登出，**Then** 设备 B 在 30 秒内显示"登录已失效，请重新登录"。
- **AC-3**（关联 US-3）：**Given** 管理员在后台选中某用户，**When** 点击"强制下线全部 session"，**Then** 该用户全部活跃 session 立即失效。
- **AC-4**（关联 US-4）：**Given** 任一 token 签发或 refresh，**When** 完成签发，**Then** 审计日志含 `user_id` / `device_id` / `issued_at` / `expires_at`。
