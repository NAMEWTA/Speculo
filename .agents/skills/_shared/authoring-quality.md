# Speculo Authoring Quality Model

本模型将 [writing-great-skills](../../../template/vendor/matt-pocock/productivity/writing-great-skills/SKILL.md) 的方法用于 Speculo 资产维护。需要术语精确定义时再读取同目录 [GLOSSARY.md](../../../template/vendor/matt-pocock/productivity/writing-great-skills/GLOSSARY.md)；vendor 是只读事实源。

## 根本目标

以**可预测性**衡量资产：每次运行遵循相同过程，而不是强求相同输出。

## 设计杠杆

- **主导词**：用模型已有的紧凑概念锚定 description、步骤和完成标准；不要用多句同义重述。
- **信息层级**：入口只保留每条分支都需要的步骤；分支细节放到有明确触发条件的直接指针后。
- **就近放置**：定义、规则、边界和完成标准放在使用它们的位置附近。
- **完成标准**：每个工作单元都以可检查条件结束；关键步骤要求穷尽，而不是只产出一个样例。
- **按调用拆分**：只有能力有独立触发词或多个调用方时才做 skill。
- **按顺序拆分**：只有可见后续步骤确实诱发过早完成时才隐藏后续阶段。
- **单一事实来源**：共享契约放在 `docs/` 或唯一 reference，调用方只建立上下文指针。

## 修剪

逐句执行两个判断：这句仍影响行为吗；它相对模型默认行为改变了什么吗。删除重复、沉积和无效操作，用正面目标描述期望行为；只有无法正面表达的安全边界才使用禁令。

## 完成标准

- 每条运行分支都能从 description 和入口可靠到达所需材料。
- 每项共享规则只有一个权威位置。
- 所有步骤都有可检查、关键处穷尽的完成标准。
- 入口中没有未触发分支的细节、同义重复或无效操作。
