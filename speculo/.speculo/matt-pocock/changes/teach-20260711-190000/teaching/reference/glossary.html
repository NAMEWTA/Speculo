术语表

Speculo Matt-Pocock Workflow 的规范语言

Workflow 层

WORKFLOW.md：工作流主定义文件。YAML frontmatter 声明元数据，正文嵌入 XML 块描述运行时上下文、持久化、路由、进入协议、依赖、状态扩展和转移规则。 Avoid: 编排文件、配置中心

路由 (Route)：用户意图到 vendor 技能执行的映射路径。包含路由 ID、优先级（order）、触发条件（when）、目标技能（skill）和产物路径（artifact）。一条路由可包含多条子路由。 Avoid: 命令、技能入口

进入协议 (Entry Protocol)：工作流的启动序列——select-change → route → lazy-config 三步。确保上下文就绪后才激活目标技能。 Avoid: 初始化流程、启动步骤

根别名 (Root Alias)：由 <runtime-context> 定义的逻辑名称，映射到 workspace.json 中的物理路径。workflow 所有 XML 块通过别名引用路径，不硬编码。 Avoid: 路径变量、路径符号

activation：技能激活模式。required = 原样执行（用于基础设施技能），adapted = 路径适配执行（用于业务技能）。 Avoid: 调用模式、执行方式

路径适配 (Path Adaptation)：adapted 技能激活时，将技能的默认文件系统路径（当前目录、.scratch/、docs/agents/、/tmp）重定向到 state namespace 中 artifact 声明的路径。 Avoid: 路径替换、路径重定向

lazy-config：进入协议 Phase 3——仅当目标路由缺少必要配置（tracker、triage labels、domain 文档）时才触发，且只补齐必需的配置项。 Avoid: 初始化配置、首次设置

State 层

change：工作流的一次执行单元。一个 change 记录从用户意图到交付完成的完整 trace，包含所有路由产物的子目录。生命周期：创建 → 活跃 → 完成门禁 → 归档。 Avoid: 任务、会话、工作单元

status.json：工作流的运行时状态文件。骨架字段（workflow、current_route、route_history、active、current_change）加 state-schema 声明的扩展字段。 Avoid: 状态配置、运行配置

store：<persistence> 中声明的持久化存储单元。由 id、role、kind（file/directory）、path 和 create 模式（initialize/lazy/existing-only）定义。 Avoid: 存储目录、数据目录

completion-gate：finalize 命令触发的完成门禁检查。验证所有 phase 已完成、无未提交代码、无未解决 external_refs、无临时原型残留后，才允许 change 归档。 Avoid: 完成检查、结束条件

external_refs：status.json 的扩展字段。记录本 workflow 产生的外部副作用的引用（如 GitHub issue URL、PR 链接），用于审计和恢复。 Avoid: 外部链接、副作用记录

artifact：路由声明中指定的产物路径模板。技能的所有文件写入经路径适配后落到此路径下。 Avoid: 输出路径、产物目录

legacy-config：create="existing-only" 的 store，用于兼容旧版 .config/ 目录格式。只读不写，存在则读取，不存在静默跳过。 Avoid: 旧配置、兼容目录

Vendor 层

vendor skill：Matt Pocock 的原始技能定义。每个技能一个目录，包含 SKILL.md 加可选参考文件。分用户调用和模型调用两类。 Avoid: 原始技能、上游技能

ask-matt：技能路由器。读取所有路由的触发条件，按优先级匹配用户意图，在歧义时一次只澄清一个决策。 Avoid: 路由引擎、意图匹配器

主流程 (Main Flow)：从 idea-to-delivery 路由的完整链路：grill → to-spec → to-tickets → implement。大多数工作的默认路径。 Avoid: 默认流程、标准路径

入口匝道 (On-ramp)：汇入主流程的起始场景路由：triage（分类）、diagnosing-bugs（诊断）、wayfinder（探索规划）。 Avoid: 入口路由、起点

原型 (Prototype)：一次性程序，回答单个设计问题。构建后保留答案，删除代码。属于模型调用技能。 Avoid: 实验代码、探索代码

Teaching 域

Mission：用户学习某个主题的原因和成功标准。所有教学决策应追溯到此。位于教学工作区根目录的 MISSION.md。 Avoid: 学习目标、课程目标

最近发展区 (Zone of Proximal Development)：用户当前能力与潜在能力之间的区间——既不太容易（无聊）也不太困难（挫败）。通过阅读 learning-records 和对照 mission 来确定。 Avoid: 学习水平、难度等级

合意难度 (Desirable Difficulty)：恰好的难度——足够挑战以建立长期记忆（存储强度），但不至于过高导致认知超载。用于技能练习设计。 Avoid: 最佳难度、适当挑战
